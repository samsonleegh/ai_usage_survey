Executive Summary

This report presents findings from an AI usage survey conducted across the Data Science team, capturing responses from 16 team members. The survey assessed self-reported AI knowledge, frequency of usage, tools adopted, current and desired use cases, barriers to adoption, and skill gaps.

The team shows strong AI adoption, but a clear gap exists between current capabilities and aspirations — particularly in machine learning tasks, data analysis automation, and workflow optimization. Key Observations
  1. ML-like tasks show the largest relative gap: Only 31% currently use AI for ML tasks, but 56% want to. Six respondents explicitly flagged this as a gap category.
  2. Workflow automation is the top desired use case: 63% want AI-powered automation (up from 44%), covering note-taking, meeting summarisation, and scheduling.
  3. Brainstorming is fully saturated: 100% already use AI for brainstorming. The team has matured past basic conversational use and wants production-oriented capabilities.
  4. Internal knowledge retrieval remains important: 50% desire better AI-powered access to internal docs, wikis, and tables — where tools like Tika and Aily could add value.
Section 04
Recommended Workshops & Actions

Four workshops directly address the identified skill gaps and barriers. Each is mapped to respondent demand.

Workshop 1: Internal AI Tooling Deep Dive

13 of 16 respondents recommended (81%) — Addresses tool overwhelm & awareness

Hands-on walkthrough of Tika, Trae, Mira, Aily, and Magibook with practical exercises. Participants select 1–2 tools to test on upcoming projects.

→ Action: Create a curated "Internal AI Tool Selector Guide" one-pager mapping each tool to ideal use cases.

Workshop 2: Machine Learning Tasks with LLMs

6 respondents recommended (38%) — Addresses the #1 gap category

Practical introduction to using LLMs for classification prompts, content labelling, video description, and small clustering pilots. Emphasis on prompt engineering for structured outputs.

→ Action: Pair beginners with advanced users (Nick, Anushka) for 1–2 week pilot sprints on real labelling tasks.

Workshop 3: AI for Data Analysis & Insights

5 respondents recommended (31%) — Addresses EDA & RCA automation needs

Using AI to support exploratory data analysis, generate quick insight drafts from dashboards, and automate root cause analysis workflows.

→ Action: Develop 2–3 reusable prompt templates for common data analysis patterns (RCA, trend analysis, anomaly summaries).

Workshop 4: Advanced Workflow Automation

5 respondents recommended (31%) — Highest desired use case at 63%

Identify 1–2 routine workflows per participant and build AI-powered automation using agents or scripting.

→ Action: Run a "Workflow Hackathon" where teams prototype one automated workflow each.
Additional Recommendations
  1. Leverage power users as champions: Nick Jenkins and Anushka Kapur (both 9/10) should co-facilitate workshops and share best practices from advanced projects.
  2. Publish an Internal AI Tool Directory: Address the 50% who don't know what tools are available with a living document listing each tool, its purpose, and example use cases.
  3. Run quarterly pulse surveys: Track gap closure, knowledge scores, and internal tool adoption to measure workshop impact.
  4. Create a shared prompt library: Establish a team-wide repository of proven prompts for code generation, data analysis, and ML tasks to reduce the learning curve.
